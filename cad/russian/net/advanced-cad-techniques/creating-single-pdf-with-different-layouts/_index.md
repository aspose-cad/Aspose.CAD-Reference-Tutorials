# Как создать единый PDF с разными макетами – руководство Aspose.CAD

## Введение

Если вам нужно **создать единый PDF**‑файл, содержащий несколько макетов CAD, вы попали по адресу. В этом руководстве мы пошагово пройдем все необходимые действия для преобразования чертежей CAD в один PDF‑документ, обрабатывая несколько макетов. Вы увидите, как Aspose.CAD для .NET упрощает **конвертацию CAD в PDF**, **экспорт DWG в PDF** и **сохранение CAD как PDF** всего несколькими строками кода на C#.

## Быстрые ответы
- **Что покрывает данное руководство?** Генерацию одного PDF, включающего несколько макетов CAD.  
- **Какая библиотека используется?** Aspose.CAD для .NET.  
- **Нужна ли лицензия?** Для оценки достаточно бесплатной пробной версии; для продакшна требуется лицензия.  
- **Поддерживаемые форматы CAD?** DWG, DXF, DGN и многие другие.  
- **Типичное время реализации?** Около 10‑15 минут для базового преобразования.

## Что означает «create single pdf» в контексте CAD?

Создание единого PDF подразумевает взятие одного исходного файла CAD, который может содержать несколько определений макетов (paper spaces), и объединение каждого макета в отдельные страницы одного PDF‑документа. Это особенно полезно для архитектурных планов, инженерных схем или любой ситуации, когда клиент ожидает единый PDF‑пакет.

## Почему стоит использовать Aspose.CAD для этой задачи?

- **Без внешних зависимостей** – чистый .NET, без необходимости установки AutoCAD.  
- **Полный контроль над растеризацией** – можно задать размер страницы, DPI и пользовательские размеры макета.  
- **Высококачественная визуализация** – векторные данные сохраняются там, где это возможно, обеспечивая чёткий результат.  
- **Готовность к пакетной обработке** – тот же код можно разместить в цикле для автоматической обработки множества чертежей.

## Импорт пространств имён

В вашем C#‑проекте подключите необходимые пространства имён, чтобы воспользоваться функциональностью Aspose.CAD для .NET:

```csharp
using Aspose.CAD.FileFormats.Cad;
```

## Пошаговое руководство

### Шаг 1: Загрузка CAD‑изображения

Сначала загрузите исходный файл CAD (например, чертёж DWG). Класс `CadImage` предоставляет доступ ко всем макетам внутри файла.

```csharp
// The path to the documents directory.
string MyDir = "Your Document Directory";

using (CadImage cadImage = (CadImage)Image.Load(MyDir + "City skyway map.dwg"))
{
    // Your code for Step 1 goes here
}
```

### Шаг 2: Настройка параметров растеризации

Определите, как каждый макет будет растеризоваться. Можно задать размер страницы по умолчанию, а затем переопределить его для конкретных макетов с помощью `LayoutPageSizes`.

```csharp
CadRasterizationOptions rasterizationOptions = new CadRasterizationOptions();
rasterizationOptions.PageWidth = 1000;
rasterizationOptions.PageHeight = 1000;

// Custom sizes for several layouts
rasterizationOptions.LayoutPageSizes.Add("ANSI C Plot", new SizeF(500, 1000));
rasterizationOptions.LayoutPageSizes.Add("8.5 x 11 Plot", new SizeF(1000, 100));
```

> **Совет:** При необходимости более детализированного вывода увеличьте DPI (`rasterizationOptions.Resolution`).

### Шаг 3: Определение параметров PDF

Обёрните настройки растеризации в объект `PdfOptions`. Это указывает Aspose.CAD напрямую рендерить данные CAD в поток PDF.

```csharp
PdfOptions pdfOptions = new PdfOptions() { VectorRasterizationOptions = rasterizationOptions };
```

### Шаг 4: Сохранение в PDF

Наконец, вызовите `Save` у экземпляра `CadImage`, передав имя целевого PDF‑файла и подготовленные параметры. Библиотека сгенерирует единый PDF, содержащий страницу для каждого настроенного макета.

```csharp
cadImage.Save(MyDir + "singlePDF_out.pdf", pdfOptions);
```

После выполнения этого вызова у вас будет PDF, объединяющий макеты «ANSI C Plot» и «8.5 x 11 Plot» в один целостный документ.

## Распространённые проблемы и их решение

| Проблема | Причина | Решение |
|----------|---------|---------|
| Макеты отсутствуют в PDF | `LayoutPageSizes` не определён для имени макета | Проверьте точные имена макетов в файле CAD (учитывается регистр). |
| Низкое качество вывода | DPI по умолчанию = 96 | Установите `rasterizationOptions.Resolution = 300` (или выше) перед сохранением. |
| Файл не найден | Неправильный путь `MyDir` | Используйте `Path.Combine` для построения кроссплатформенных путей. |
