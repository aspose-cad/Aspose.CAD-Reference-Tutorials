# Создание PDF из CAD с отслеживанием с помощью Aspose.CAD для .NET

## Введение

В современных .NET‑приложениях **создание PDF из CAD** файлов является распространённой задачей — будь то необходимость поделиться проектами с нетехническими заинтересованными сторонами или архивировать чертежи в переносимом формате. Aspose.CAD для .NET упрощает эту конверсию и одновременно предоставляет функцию **tracking**, позволяющую контролировать процесс рендеринга и собирать диагностическую информацию. В этом руководстве мы пройдём весь процесс: загрузка DXF‑файла, настройка размера страницы, конвертация в PDF и включение отслеживания для получения полной видимости работы рендеринга.

## Быстрые ответы
- **Можно ли конвертировать DXF в PDF с помощью Aspose.CAD?** Да, библиотека поддерживает конвертацию DXF‑в‑PDF из коробки.  
- **Как задать размер страницы CAD при конвертации?** Используйте `CadRasterizationOptions.PageWidth` и `PageHeight`.  
- **Обязательно ли включать отслеживание?** Нет, но оно предоставляет ценные диагностические данные для больших или сложных чертежей.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **Нужна ли лицензия для продакшн?** Требуется коммерческая лицензия; доступна бесплатная пробная версия.

## Что означает «создание PDF из CAD»?
Создание PDF из CAD подразумевает рендеринг CAD‑чертежа (например, DXF, DWG) в растровый или векторный PDF‑документ. Этот процесс сохраняет визуальную точность и предоставляет формат, который можно открыть практически на любом устройстве без специализированного CAD‑ПО.

## Зачем включать отслеживание при рендеринге CAD?
Отслеживание даёт возможность в реальном времени наблюдать за работой движка рендеринга — это полезно для отладки, оптимизации производительности и аудита. При включённом отслеживании Aspose.CAD записывает каждый шаг рендеринга, помогая выявить узкие места или ошибки в крупных проектах.

## Предварительные требования

- **Aspose.CAD for .NET** – скачайте его [здесь](https://releases.aspose.com/cad/net/).  
- **Среда разработки** – Visual Studio (любая современная версия) с поддержкой .NET.  
- **Пример CAD‑файла** – DXF‑файл, например `conic_pyramid.dxf`, который вы будете конвертировать и отслеживать.

## Пошаговое руководство

### Шаг 1: Установите каталог документа (set page size CAD)

```csharp
string MyDir = "Your Document Directory";
string sourceFilePath = MyDir + "conic_pyramid.dxf";
```

Замените **Your Document Directory** на абсолютный путь, где находится ваш DXF‑файл. Здесь же позже можно будет скорректировать размеры страницы через параметры растеризации.

### Шаг 2: Загрузите CAD‑файл

```csharp
using (Image image = Image.Load(sourceFilePath))
{
    // Further steps will be implemented here
}
```

Метод `Image.Load` считывает CAD‑чертёж в объект `Aspose.CAD.Image`, подготавливая его к рендерингу.

### Шаг 3: Создайте параметры PDF (prepare for convert dxf to pdf)

```csharp
MemoryStream stream = new MemoryStream();
PdfOptions pdfOptions = new PdfOptions();
```

`MemoryStream` позволяет держать сгенерированный PDF в памяти, а `PdfOptions` содержит все настройки, специфичные для PDF.

### Шаг 4: Настройте параметры растеризации (set page size CAD)

```csharp
CadRasterizationOptions cadRasterizationOptions = new CadRasterizationOptions();
pdfOptions.VectorRasterizationOptions = cadRasterizationOptions;
cadRasterizationOptions.PageWidth = 800;
cadRasterizationOptions.PageHeight = 600;
```

Здесь задаётся размер выходной страницы (`PageWidth` и `PageHeight`). Отрегулируйте эти значения в соответствии с требуемыми размерами PDF — это основная часть требования **set page size CAD**.

### Шаг 5: Сохраните отрендеренное изображение (complete the create pdf from cad workflow)

```csharp
image.Save(stream, pdfOptions);
```

Вызов `Save` записывает отрендеренное содержимое в поток памяти, используя ранее сконфигурированные параметры PDF. На этом этапе у вас есть PDF‑представление исходного CAD‑файла, а информация об отслеживании автоматически собрана библиотекой.

## Распространённые проблемы и решения

| Проблема | Почему происходит | Решение |
|----------|-------------------|---------|
| **Пустой PDF‑файл** | Параметры растеризации не привязаны к `PdfOptions` | Убедитесь, что установлено `pdfOptions.VectorRasterizationOptions = cadRasterizationOptions;`. |
| **Неправильный размер страницы** | `PageWidth`/`PageHeight` оставлены со значениями по умолчанию | Явно задайте оба свойства перед сохранением. |
| **Замедление производительности на больших DXF** | Логи отслеживания могут быть слишком подробными | Отключите или ограничьте отслеживание в продакшн, изменив настройки `Aspose.CAD.Logging`. |
