# Конвертировать DWG в PDF (PDF/A1a и A1b) с помощью Aspose.CAD for Java

## Введение

В современных инженерных и дизайнерских процессах **convert DWG to PDF** является ежедневной необходимостью для обмена, архивирования и соответствия отраслевым стандартам документов. PDF/A‑1a и PDF/A‑1b — самые широко принятые стандарты архивирования, гарантирующие, что ваши чертежи будут отображаться одинаково на любой системе. Aspose.CAD for Java делает эту конвертацию простой, надёжной и полностью программируемой. В этом руководстве вы узнаете, как именно конвертировать файлы DWG в PDF, соответствующие PDF/A, используя всего несколько строк кода на Java.

## Быстрые ответы
- **Какая библиотека выполняет конвертацию?** Aspose.CAD for Java  
- **Какие стандарты PDF/A поддерживаются?** PDF/A‑1a и PDF/A‑1b  
- **Нужна ли лицензия для тестирования?** Да — временная лицензия доступна от Aspose.  
- **Какая минимальная версия Java?** Рекомендуется Java 8 или выше.  
- **Можно ли пакетно обрабатывать несколько файлов DWG?** Конечно — повторите шаги для каждого файла или выполните цикл по папке.

## Что такое «конвертировать DWG в PDF» и почему это важно?
Конвертация чертежа DWG в PDF создаёт документ, который можно просматривать в любой системе, сохраняя при этом векторную точность. При выборе соответствия PDF/A‑1a или PDF/A‑1b файл также встраивает цветовые профили, шрифты и метаданные, необходимые для долгосрочного архивирования, что критично для регуляторных заявок, строительной документации и поставки клиентам.

## Почему использовать Aspose.CAD for Java?
- **Полная поддержка версий DWG** — от старого R12 до последних релизов.  
- **Отсутствие внешних зависимостей** — библиотека работает сразу, без необходимости в CAD‑программах.  
- **Тонкая настройка** — можно задать параметры растеризации, DPI, размер страницы и соответствие PDF/A.  
- **Высокая производительность** — подходит для пакетной обработки тысяч чертежей.

## Требования

Перед началом убедитесь, что у вас есть:

- Среда разработки Java (JDK 8 или новее) с вашей любимой IDE.  
- Библиотека Aspose.CAD for Java — скачайте её по [download link](https://releases.aspose.com/cad/java/).  
- Папка, содержащая DWG‑чертежи, которые нужно конвертировать.

## Импорт пространств имён

Сначала импортируйте необходимые классы. Размещение импортов в начале файла упрощает чтение и поддержку кода.

```java
import com.aspose.cad.Image;

import com.aspose.cad.imageoptions.CadRasterizationOptions;
import com.aspose.cad.imageoptions.PdfCompliance;
import com.aspose.cad.imageoptions.PdfDocumentOptions;
import com.aspose.cad.imageoptions.PdfOptions;
```

## Шаг 1: Установите каталог ресурсов

Определите путь, где находятся ваши файлы DWG. Измените строку, чтобы она указывала на ваш реальный каталог.

```java
String dataDir = "Your Document Directory" + "DWGDrawings/";
```

## Шаг 2: Загрузите файл DWG

Используйте `Image.load` для чтения файла DWG в память. Этот объект позже будет сохранён как PDF.

```java
String srcFile = dataDir + "Bottom_plate.dwg";
Image objImage = Image.load(srcFile);
```

## Шаг 3: Создайте параметры PDF

Создайте экземпляр `PdfOptions` и присоедините объект `CadRasterizationOptions`. Параметры растеризации позволяют контролировать, как векторные данные будут отрисованы внутри PDF.

```java
PdfOptions pdfOptions = new PdfOptions();
pdfOptions.setVectorRasterizationOptions(new CadRasterizationOptions());
```

## Шаг 4: Установите основные параметры PDF (соответствие)

Здесь вы указываете Aspose.CAD уровень соответствия PDF/A, который вам нужен. В примере используется PDF/A‑1a.

```java
pdfOptions.setCorePdfOptions(new PdfDocumentOptions());
pdfOptions.getCorePdfOptions().setCompliance(PdfCompliance.PdfA1a);
```

## Шаг 5: Сохраните PDF с соответствием PDF/A‑1a

Теперь запишите PDF‑файл на диск. Полученный файл будет соответствовать PDF/A‑1a, готовый к архивированию.

```java
objImage.save(dataDir + "Saved1.pdf", pdfOptions);
```

## Шаг 6: Измените соответствие на PDF/A‑1b и сохраните снова

Если нужен PDF/A‑1b, просто измените флаг соответствия и сохраните второй файл.

```java
pdfOptions.getCorePdfOptions().setCompliance(PdfCompliance.PdfA1b);
objImage.save(dataDir + "Saved.pdf", pdfOptions);
```

> **Pro tip:** Оберните логику конвертации в переиспользуемый метод, чтобы вызывать её для каждого DWG в папке, уменьшая дублирование кода и повышая поддерживаемость.

## Распространённые проблемы и решения

- **Отсутствуют шрифты или символы** — убедитесь, что DWG‑файл содержит все необходимые шрифты, или установите `CadRasterizationOptions.setEmbedFonts(true)`.  
- **Большой размер файла** — уменьшите DPI в параметрах растеризации или включите сжатие через `PdfDocumentOptions.setCompress(true)`.  
- **Лицензия не найдена** — примените временную или постоянную лицензию перед конвертацией; иначе будет водяной знак.
