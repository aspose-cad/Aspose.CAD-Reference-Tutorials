## Réponses rapides
- **Quel est le but principal d'Aspose.CAD pour .NET ?** Lire, modifier et convertir les fichiers CAD et BIM (y compris DWG, DWFX, DWF) de manière programmatique.  
- **Puis-je ouvrir directement un fichier DWFX en C# ?** Oui – Aspose.CAD fournit une API simple pour charger et accéder au contenu DWFX.  
- **Ai-je besoin d'une licence pour le développement ?** Une licence d'évaluation gratuite fonctionne pour les tests ; une licence commerciale est requise pour la production.  
- **Quelles versions de .NET sont prises en charge ?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.  
- **L'ajout de texte à un fichier DWG est-il pris en charge ?** Absolument – vous pouvez insérer des entités de texte en utilisant l'API CAD.

## Questions fréquemment posées

**Q : Comment ouvrir un fichier DWFX en C# ?**  
R : Utilisez `Aspose.CAD.Image.Load("file.dwfx")` pour charger le fichier, puis accédez à ses pages ou entités selon les besoins.

**Q : Puis-je lire un fichier DWG avec C# sans le convertir au préalable ?**  
R : Oui – Aspose.CAD peut charger directement les fichiers DWG, vous permettant d'interroger les calques, les entités et les métadonnées.

**Q : Quelle est la façon la plus simple d'ajouter du texte à un dessin DWG ?**  
R : Créez une entité `CadText`, définissez sa `Value` et sa position, puis ajoutez‑la à la collection du modèle d'espace du dessin.

**Q : Existe‑t‑il des limites de taille lors de la manipulation de gros fichiers DWFX ou DWG ?**  
R : La bibliothèque gère efficacement les gros fichiers, mais vous devez vous assurer de disposer de suffisamment de mémoire et envisager le streaming de grandes images raster.

**Q : Ai‑je besoin d’une licence séparée pour chaque format CAD ?**  
R : Non – une seule licence Aspose.CAD couvre tous les formats pris en charge, y compris DWG, DWFX, DWF, DXF, et plus.
