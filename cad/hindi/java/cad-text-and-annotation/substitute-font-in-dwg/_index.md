# Java में DWG फ़ाइल लोड करना और Aspose.CAD के साथ फ़ॉन्ट बदलना

## परिचय

यदि आपको Java अनुप्रयोगों में **convert dxf to dwg** करने की आवश्यकता है और अपने ड्रॉइंग्स को एक परिष्कृत रूप देना है, तो फ़ॉन्ट बदलना एक त्वरित समाधान है। Aspose.CAD for Java के साथ आप डिफ़ॉल्ट टेक्स्ट स्टाइल को किसी भी सिस्टम‑स्थापित फ़ॉन्ट से बदल सकते हैं, जिससे हर एनोटेशन ठीक वैसा ही दिखे जैसा आप चाहते हैं। इस ट्यूटोरियल में हम पूरी प्रक्रिया को समझेंगे—Java में DWG फ़ाइल लोड करने से लेकर फ़ॉन्ट बदलने के लिए `setPrimaryFontName` मेथड का उपयोग करने तक।

## त्वरित उत्तर
- **मुझे कौनसी लाइब्रेरी चाहिए?** Aspose.CAD for Java.  
- **क्या मैं Java में DWG फ़ाइल लोड कर सकता हूँ?** हाँ, बस DWG पाथ पर `Image.load()` कॉल करें।  
- **फ़ॉन्ट बदलने वाली मेथड कौनसी है?** `setPrimaryFontName`.  
- **उत्पादन के लिए लाइसेंस चाहिए?** हाँ, एक व्यावसायिक लाइसेंस आवश्यक है।  
- **क्या बैच प्रोसेसिंग संभव है?** बिल्कुल – एक ही कोड से कई फ़ाइलों को लूप करें।

## **convert dxf to dwg** क्या है?

“Convert dxf to dwg” का अर्थ है DXF (Drawing Exchange Format) फ़ाइल को कई CAD अनुप्रयोगों द्वारा उपयोग किए जाने वाले मूल DWG फ़ॉर्मेट में बदलना। रूपांतरण से आप व्यापक रूप से समर्थित DXF ड्रॉइंग को DWG कार्यप्रवाह में ला सकते हैं, और फिर Aspose.CAD का उपयोग करके उन्नत स्टाइलिंग—जैसे फ़ॉन्ट प्रतिस्थापन—को लागू कर सकते हैं।

## DWG फ़ाइल में फ़ॉन्ट बदलना क्यों आवश्यक है?

- **संगति:** यह सुनिश्चित करता है कि सभी सहयोगी एक ही फ़ॉन्ट देखें, चाहे उनका स्थानीय फ़ॉन्ट सेटअप कुछ भी हो।  
- **पठनीयता:** कुछ डिफ़ॉल्ट CAD फ़ॉन्ट स्क्रीन पर पढ़ने में कठिन होते हैं; Arial जैसे साफ़ फ़ॉन्ट में बदलने से स्पष्टता बढ़ती है।  
- **ब्रांडिंग:** तकनीकी ड्रॉइंग्स को कॉरपोरेट स्टाइल गाइड्स के साथ संरेखित करता है।  

## सामान्य उपयोग केस

| परिदृश्य | क्यों महत्वपूर्ण है |
|----------|-------------------|
| **पुराने DXF ड्रॉइंग्स का बैच रूपांतरण** | आप इन्हें DWG में बदल सकते हैं और एक ही पास में कॉरपोरेट फ़ॉन्ट लागू कर सकते हैं। |
| **क्लाइंट प्रस्तुतियों के लिए ड्रॉइंग्स तैयार करना** | संगत, पढ़ने योग्य फ़ॉन्ट दस्तावेज़ों को पेशेवर बनाते हैं। |
| **स्वचालित CAD पाइपलाइन** | फ़ॉन्ट प्रतिस्थापन को एम्बेड करने से मैन्युअल पोस्ट‑प्रोसेसिंग चरण समाप्त हो जाते हैं। |

## पूर्वापेक्षाएँ

- **Java Development Kit (JDK)** – कोई भी नवीनतम संस्करण (8+ अनुशंसित)।  
- **Aspose.CAD for Java** – [website](https://releases.aspose.com/cad/java/) से डाउनलोड करें।  
- **Sample DWG/DXF file** – वह ड्रॉइंग जिसे आप संशोधित करना चाहते हैं (परीक्षण के लिए आप कोई भी DWG या DXF फ़ाइल उपयोग कर सकते हैं)।

## नेमस्पेस इम्पोर्ट करें

अपने Java प्रोजेक्ट में, Aspose.CAD के साथ काम करने के लिए आवश्यक क्लासेस इम्पोर्ट करें। ये इम्पोर्ट्स आपको इमेज लोडिंग, CAD‑विशिष्ट ऑब्जेक्ट्स, और स्टाइल मैनिपुलेशन तक पहुँच प्रदान करते हैं।

```java
import com.aspose.cad.Image;

import com.aspose.cad.fileformats.cad.CadImage;
import com.aspose.cad.fileformats.cad.cadtables.CadStyleTableObject;
```

## फ़ॉन्ट बदलने के लिए चरण‑दर‑चरण गाइड

### चरण 1: अपनी DWG फ़ाइल लोड करें (load dwg file java)

पहले, API को अपनी DWG (या DXF) फ़ाइल के स्थान की ओर इंगित करें और इसे `CadImage` ऑब्जेक्ट में लोड करें।

```java
// The path to the resource directory.
String dataDir = "Your Document Directory" + "CADConversion/";

String srcFile = dataDir + "conic_pyramid.dxf";
CadImage cadImage = (CadImage) Image.load(srcFile);
```

> **प्रो टिप:** यदि आप सीधे DWG फ़ाइलों के साथ काम कर रहे हैं, तो `srcFile` वेरिएबल में `.dxf` एक्सटेंशन को `.dwg` से बदल दें।

### चरण 2: स्टाइल टेबल पर इटररेट करें और प्राइमरी फ़ॉन्ट नाम सेट करें

प्रत्येक CAD ड्रॉइंग में स्टाइल ऑब्जेक्ट्स का संग्रह होता है। उन पर लूप करें और फ़ॉन्ट बदलने के लिए `setPrimaryFontName` मेथड (API कॉल जो **प्राइमरी फ़ॉन्ट नाम सेट करती है**) का उपयोग करें।

```java
for(Object style : cadImage.getStyles())
{
    // Set the font name
    ((com.aspose.cad.fileformats.cad.cadtables.CadStyleTableObject)style).setPrimaryFontName("Arial");
}
```

आप `"Arial"` को किसी भी फ़ॉन्ट से बदल सकते हैं जो कोड चलाने वाली मशीन पर स्थापित है।

### चरण 3: संशोधित ड्रॉइंग को सहेजें

फ़ॉन्ट अपडेट करने के बाद, बदलावों को नई DWG फ़ाइल में लिखें (या मूल फ़ाइल को ओवरराइट करें)।

```java
cadImage.save(dataDir + "output.dwg", new DwgOptions());
```

सहेजी गई फ़ाइल अब आपके निर्दिष्ट फ़ॉन्ट का उपयोग करती है, और सभी टेक्स्ट एनोटेशन उस टाइपफ़ेस में रेंडर होंगे।

## सामान्य समस्याएँ और समाधान

| समस्या | समाधान |
|--------|--------|
| **फ़ॉन्ट लागू नहीं हुआ** | सुनिश्चित करें कि फ़ॉन्ट होस्ट OS पर स्थापित है और वर्तनी बिल्कुल मिलती है। |
| **`Image.load` अपवाद फेंकता है** | फ़ाइल पाथ सही है और फ़ाइल समर्थित DWG/DXF फ़ॉर्मेट में है, यह सुनिश्चित करें। |
| **बड़ी फ़ाइलों पर प्रदर्शन में गिरावट** | फ़ाइलों को अलग थ्रेड में प्रोसेस करें या बैच में चलाएँ ताकि UI ब्लॉकिंग से बचा जा सके। |

## अक्सर पूछे जाने वाले प्रश्न

**Q: क्या `setPrimaryFontName` मेथड केवल टेक्स्ट एंटिटीज़ को प्रभावित करती है?**  
A: यह स्टाइल टेबल के प्राइमरी फ़ॉन्ट को अपडेट करती है, जो बदले में उस स्टाइल को संदर्भित सभी टेक्स्ट ऑब्जेक्ट्स को प्रभावित करता है।

**Q: क्या मैं ऐसा कस्टम फ़ॉन्ट उपयोग कर सकता हूँ जो सिस्टम पर स्थापित नहीं है?**  
A: Aspose.CAD ऑपरेटिंग सिस्टम के फ़ॉन्ट रजिस्ट्री पर निर्भर करता है। कस्टम फ़ॉन्ट उपयोग करने के लिए, उसे कोड चलाने वाली मशीन पर स्थापित करें।

**Q: क्या केवल एक लेयर के लिए फ़ॉन्ट बदलना संभव है?**  
A: हां, आप `setPrimaryFontName` कॉल करने से पहले लेयर नाम से स्टाइल्स को फ़िल्टर कर सकते हैं।

**Q: DWG 2022 फ़ाइलों के लिए Aspose.CAD का कौन सा संस्करण आवश्यक है?**  
A: नवीनतम रिलीज़ (2025 तक) पूरी तरह से DWG 2022 को सपोर्ट करती है। विशिष्ट फ़ॉर्मेट सपोर्ट के लिए हमेशा रिलीज़ नोट्स देखें।

**Q: डेवलपमेंट एनवायरनमेंट में लाइसेंसिंग कैसे संभालें?**  
A: टेस्टिंग के लिए एक अस्थायी इवैल्यूएशन लाइसेंस उपयोग करें। प्रोडक्शन के लिए, `License.setLicense("Aspose.Total.Java.lic");` का उपयोग करके अपनी खरीदी हुई लाइसेंस फ़ाइल एम्बेड करें।
