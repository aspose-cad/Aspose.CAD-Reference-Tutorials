# CAD से PDF बनाएं – ऑटो लेआउट स्केलिंग के साथ Aspose.CAD Java

## परिचय

यदि आपको **CAD फ़ाइलों से PDF जल्दी और बिल्कुल सही स्केलिंग के साथ बनाना** है, तो Aspose.CAD for Java आपके लिए तैयार है। ऑटो लेआउट स्केलिंग स्वचालित रूप से लेआउट आयामों को समायोजित करता है ताकि परिणामी PDF मूल CAD पेज आकार की परवाह किए बिना ठीक वैसा ही दिखे जैसा आप चाहते हैं। इस ट्यूटोरियल में हम पूरी प्रक्रिया को चरण‑दर‑चरण देखेंगे—DXF फ़ाइल लोड करने से लेकर PDF निर्यात करने तक—और लाइब्रेरी की **CAD को PDF में निर्यात** क्षमताओं को उजागर करेंगे।

## त्वरित उत्तर
- **ऑटो लेआउट स्केलिंग क्या करता है?** यह रास्टराइज़ेशन के दौरान CAD लेआउट को लक्ष्य पेज आयामों में फिट होने के लिए स्वचालित रूप से आकार बदलता है।
- **मैं कौन‑से फ़ॉर्मेट बदल सकता हूँ?** Aspose.CAD द्वारा समर्थित कोई भी फ़ॉर्मेट (जैसे DXF, DWG, DWF) PDF में बदला जा सकता है।
- **उत्पादन के लिए लाइसेंस चाहिए?** हाँ, गैर‑मूल्यांकन उपयोग के लिए एक व्यावसायिक लाइसेंस आवश्यक है।
- **परिवर्तन में कितना समय लगता है?** आधुनिक हार्डवेयर पर सामान्य फ़ाइलों के लिए आमतौर पर एक सेकंड से कम।
- **क्या मैं पेज आकार बदल सकता हूँ?** हाँ, आप `CadRasterizationOptions` के माध्यम से कस्टम पेज आयाम सेट कर सकते हैं।

## “CAD से PDF बनाना” क्या है?
CAD से PDF बनाना का अर्थ है एक वेक्टर‑आधारित इंजीनियरिंग ड्राइंग (DXF, DWG आदि) को रास्टराइज़ करके PDF दस्तावेज़ में बदलना। PDF मूल ड्राइंग की दृश्य सटीकता को बनाए रखता है और किसी भी प्लेटफ़ॉर्म पर आसानी से देखा जा सकता है।

## ऑटो लेआउट स्केलिंग क्यों उपयोग करें?
- **सुसंगत आउटपुट:** सभी लेआउट बिना मैन्युअल आकार गणना के PDF पेज को भरते हैं।
- **समय बचत:** प्रत्येक ड्राइंग के लिए स्केलिंग फैक्टर को मैन्युअल रूप से समायोजित करने की आवश्यकता नहीं।
- **उच्च गुणवत्ता:** परिवर्तन के दौरान लाइन वेट और ज्यामिति की सटीकता बनी रहती है।

## पूर्वापेक्षाएँ

1. **Aspose.CAD for Java लाइब्रेरी** – नवीनतम संस्करण [डाउनलोड पेज](https://releases.aspose.com/cad/java/) से प्राप्त करें।  
2. **रिसोर्स डायरेक्टरी** – अपने मशीन पर एक फ़ोल्डर बनाकर CAD फ़ाइलें रखें; कोड में `"Your Document Directory"` को उस पाथ से बदलें।  
3. **सैंपल CAD फ़ाइल** – इस गाइड के लिए हम `conic_pyramid.dxf` उपयोग करेंगे, जो Aspose सैंपल डेटा सेट में शामिल है।

## नेमस्पेस इम्पोर्ट करें

पहले आवश्यक क्लासेज़ इम्पोर्ट करें। इससे हमें इमेज लोडिंग, रास्टराइज़ेशन और PDF निर्यात सुविधाओं तक पहुँच मिलती है।

```java
import com.aspose.cad.Image;

import com.aspose.cad.imageoptions.CadRasterizationOptions;
import com.aspose.cad.imageoptions.PdfOptions;
```

## चरण 1: CAD फ़ाइल लोड करें

स्रोत फ़ाइल को लोड करना **CAD को PDF में निर्यात** करने की प्रक्रिया का पहला कदम है।

```java
String srcFile = dataDir + "conic_pyramid.dxf";
Image image = Image.load(srcFile);
```

## चरण 2: रास्टराइज़ेशन विकल्प बनाएं

लक्ष्य पेज आयाम निर्धारित करें। यदि आप कस्टम लेआउट चाहते हैं तो इस ब्लॉक का उपयोग **CAD पेज आकार सेट** करने के लिए भी कर सकते हैं।

```java
CadRasterizationOptions rasterizationOptions = new CadRasterizationOptions();
rasterizationOptions.setPageWidth(1600);
rasterizationOptions.setPageHeight(1600);
```

## चरण 3: ऑटो लेआउट स्केलिंग सेट करें

स्वचालित स्केलिंग फीचर को सक्षम करें। यह **CAD‑to‑PDF परिवर्तन** में स्केलिंग सेट करने का मुख्य भाग है।

```java
rasterizationOptions.setAutomaticLayoutsScaling(true);
```

## चरण 4: PDF विकल्प बनाएं

रास्टराइज़ेशन सेटिंग्स को PDF निर्यात विकल्पों से लिंक करें।

```java
PdfOptions pdfOptions = new PdfOptions();
pdfOptions.setVectorRasterizationOptions(rasterizationOptions);
```

## चरण 5: PDF में निर्यात करें

अंत में, रेंडर की गई इमेज को PDF फ़ाइल के रूप में सहेजें। यह चरण **DXF को PDF में बदलने** की पूरी वर्कफ़्लो को पूरा करता है।

```java
image.save(dataDir + "result_out_.pdf", pdfOptions);
```

ऊपर बताए गए चरणों को किसी भी अतिरिक्त CAD फ़ाइल के लिए दोहराएँ जिन्हें आप प्रोसेस करना चाहते हैं।

## सामान्य समस्याएँ एवं ट्रबलशूटिंग

| लक्षण | संभावित कारण | समाधान |
|---------|--------------|-----|
| खाली PDF आउटपुट | रास्टराइज़ेशन विकल्प सेट नहीं या फ़ाइल पाथ गलत | `srcFile` पाथ जाँचें और सुनिश्चित करें `setPageWidth/Height` शून्य नहीं हैं |
| विकृत स्केलिंग | `setAutomaticLayoutsScaling` को `false` रखा गया | ऑटोमैटिक स्केलिंग सक्षम करें या मैन्युअल रूप से स्केलिंग फैक्टर गणना करें |
| लेयर्स गायब | स्रोत DXF में असमर्थित एंटिटीज़ हैं | समर्थित एंटिटी प्रकारों के लिए Aspose.CAD रिलीज़ नोट्स देखें |

## अक्सर पूछे जाने वाले प्रश्न

### Q1: क्या Aspose.CAD for Java सभी CAD फ़ाइल फ़ॉर्मेट्स के साथ संगत है?

A1: Aspose.CAD for Java विभिन्न CAD फ़ॉर्मेट्स को सपोर्ट करता है, जिसमें DWG, DXF, और DWF शामिल हैं।

### Q2: क्या मैं स्केलिंग विकल्पों को और कस्टमाइज़ कर सकता हूँ?

A2: हाँ, `CadRasterizationOptions` क्लास स्केलिंग और अन्य सेटिंग्स को फाइन‑ट्यून करने के लिए कई प्रॉपर्टीज़ प्रदान करती है।

### Q3: Aspose.CAD for Java के लिए अतिरिक्त दस्तावेज़ कहाँ मिलेंगे?

A3: विस्तृत जानकारी और उदाहरणों के लिए [डॉक्यूमेंटेशन](https://reference.aspose.com/cad/java/) देखें।

### Q4: क्या Aspose.CAD for Java का मुफ्त ट्रायल उपलब्ध है?

A4: हाँ, आप Aspose.CAD for Java की क्षमताओं को आज़माने के लिए एक [फ्री ट्रायल](https://releases.aspose.com/) ले सकते हैं।

### Q5: मैं Aspose.CAD for Java के बारे में सहायता कैसे प्राप्त करूँ या चर्चा में भाग ले सकूँ?

A5: समुदाय से जुड़ने और समर्थन प्राप्त करने के लिए [Aspose.CAD फ़ोरम](https://forum.aspose.com/c/cad/19) पर जाएँ।

**अतिरिक्त सामान्य प्रश्न**

**प्रश्न: DWG फ़ाइल को DXF की बजाय PDF में कैसे बदलूँ?**  
उत्तर: वही कोड काम करेगा; केवल `srcFile` में फ़ाइल एक्सटेंशन को `.dwg` बदल दें।

**प्रश्न: उच्च‑रिज़ॉल्यूशन PDFs के लिए कस्टम DPI सेट कर सकता हूँ?**  
उत्तर: हाँ, `rasterizationOptions.setResolution(300);` (या आवश्यक DPI) उपयोग करें।

**प्रश्न: उत्पन्न PDF में फ़ॉन्ट एम्बेड करना संभव है?**  
उत्तर: Aspose.CAD ड्राइंग को रास्टराइज़ करता है, इसलिए फ़ॉन्ट वेक्टर के रूप में रेंडर होते हैं; अलग फ़ॉन्ट एम्बेडिंग की आवश्यकता नहीं।

## निष्कर्ष

इस गाइड को फॉलो करके आप अब **CAD फ़ाइलों से PDF बनाने** के लिए Aspose.CAD for Java के साथ ऑटो लेआउट स्केलिंग का उपयोग कर सकते हैं। यह प्रक्रिया **CAD को PDF में निर्यात** वर्कफ़्लो को सरल बनाती है, स्केलिंग में निरंतरता सुनिश्चित करती है, और विकास समय बचाती है। विभिन्न पेज आकार, रिज़ॉल्यूशन और CAD फ़ॉर्मेट्स के साथ प्रयोग करने में संकोच न करें ताकि आपके प्रोजेक्ट की आवश्यकताओं को पूरा किया जा सके।
