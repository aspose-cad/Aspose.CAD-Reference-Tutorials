# Aspose.CAD for Java का उपयोग करके DWG को पढ़ना और DWG में लेआउट्स की सूची बनाना

## परिचय

यदि आप प्रोग्रामेटिक रूप से **DWG फ़ाइलें कैसे पढ़ें** के लिए एक विश्वसनीय तरीका खोज रहे हैं, तो Aspose.CAD for Java एक साफ़, क्रॉस‑प्लेटफ़ॉर्म API प्रदान करता है जो आपको ड्राइंग लोड करने और आवश्यक कोई भी जानकारी निकालने की सुविधा देता है—जैसे फ़ाइल के भीतर सभी लेआउट्स के नाम। इस चरण‑दर‑चरण ट्यूटोरियल में हम आपको **DWG को पढ़ने** और DWG (या DXF) फ़ाइल में मौजूद प्रत्येक लेआउट की सूची दिखाने का तरीका बताएँगे, ताकि आप इस क्षमता को किसी भी Java एप्लिकेशन में एकीकृत कर सकें जो CAD डेटा के साथ काम करता है।

## त्वरित उत्तर
- **कौन सी लाइब्रेरी आवश्यक है?** Aspose.CAD for Java.  
- **क्या मैं किसी भी OS पर DWG फ़ाइलें पढ़ सकता हूँ?** हाँ – Java क्रॉस‑प्लेटफ़ॉर्म है, इसलिए आप **Linux पर DWG पढ़ सकते हैं** जितना आसानी से Windows पर।  
- **क्या सैंपल चलाने के लिए लाइसेंस चाहिए?** मूल्यांकन के लिए एक मुफ्त ट्रायल काम करता है; प्रोडक्शन के लिए लाइसेंस आवश्यक है।  
- **कौन‑से CAD फ़ॉर्मेट समर्थित हैं?** DWG, DXF, DWF, और अन्य।  
- **क्या कोड Java 8+ के साथ संगत है?** बिल्कुल।

## Java में “DWG कैसे पढ़ें” क्या है?

DWG फ़ाइल पढ़ना मतलब बाइनरी CAD डेटा को एक ऑब्जेक्ट मॉडल में लोड करना है जिसे आप क्वेरी कर सकते हैं। Aspose.CAD जटिल DWG संरचना को सरल Java क्लासेज़ के पीछे छिपा देता है, जिससे आप केवल आवश्यक जानकारी—इस मामले में लेआउट नामों—पर ध्यान केंद्रित कर सकते हैं।

## DWG फ़ाइल में लेआउट्स की सूची क्यों बनाएं?

DWG में कई लेआउट्स (पेपर स्पेस, मॉडल स्पेस, कस्टम शीट्स) हो सकते हैं। लेआउट नामों को जानने से आप:

- प्रत्येक लेआउट के लिए रिपोर्ट जनरेट कर सकते हैं।  
- विशिष्ट लेआउट्स को इमेज या PDF में एक्सपोर्ट कर सकते हैं।  
- ड्राइंग्स की बैच प्रोसेसिंग को ऑटोमेट कर सकते हैं।  

## पूर्वापेक्षाएँ

कोड में डुबकी लगाने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

- **Aspose.CAD for Java लाइब्रेरी** – नवीनतम JAR को [वेबसाइट](https://releases.aspose.com/cad/java/) से डाउनलोड करें।  
- **Java विकास पर्यावरण** – JDK 8 या उससे ऊपर, और आपका पसंदीदा IDE या बिल्ड टूल।

## नेमस्पेस आयात करें

अपने Java स्रोत फ़ाइल में आवश्यक Aspose.CAD क्लासेज़ आयात करें:

```java
import com.aspose.cad.Image;

import com.aspose.cad.fileformats.cad.CadImage;
import com.aspose.cad.fileformats.cad.CadLayoutDictionary;
import com.aspose.cad.fileformats.cad.cadobjects.CadLayout;
```

## चरण 1: अपना डॉक्यूमेंट डायरेक्टरी सेट करें

```java
String dataDir = "Your Document Directory" + "CADConversion/";
```

**“Your Document Directory”** को उस पूर्ण पथ से बदलें जहाँ आपके CAD फ़ाइलें स्थित हैं। Linux पर आप `/home/user/cad/` जैसा पथ उपयोग कर सकते हैं।

## चरण 2: DWG फ़ाइल लोड करें

```java
String sourceFilePath = dataDir + "conic_pyramid.dxf";
Image image = Image.load(sourceFilePath);
CadImage cadImage = (CadImage)image;
```

`Image.load` मेथड फ़ाइल फ़ॉर्मेट को स्वचालित रूप से पहचान लेता है, इसलिए वही कोड **DWG** और **DXF** दोनों फ़ाइलों के लिए काम करता है।

## चरण 3: लेआउट्स प्राप्त करें और नाम प्रिंट करें

```java
CadLayoutDictionary layouts = cadImage.getLayouts();
for (CadLayout layout : layouts.getValues())
{
    System.out.println("Layout " + layout.getLayoutName());
}
```

यह लूप प्रत्येक लेआउट ऑब्जेक्ट पर इटरिट करता है और उसका नाम कंसोल में प्रिंट करता है—एक सरल तरीका यह सत्यापित करने का कि आपने सफलतापूर्वक **DWG पढ़ा** और लेआउट जानकारी निकाली।

## Linux पर DWG को PDF में कैसे कनवर्ट करें

यदि बाद में आपको किसी विशिष्ट लेआउट को PDF में बदलना हो, तो Aspose.CAD लेआउट को इमेज में रेंडर कर सकता है और फिर आप Aspose.PDF (या कोई अन्य PDF लाइब्रेरी) का उपयोग करके उस इमेज को PDF दस्तावेज़ में एम्बेड कर सकते हैं। कनवर्ज़न कोड Linux पर भी समान रहता है क्योंकि API पूरी तरह से Java है।

## सामान्य समस्याएँ एवं टिप्स

- **गलत फ़ाइल पथ** – सुनिश्चित करें कि `dataDir` के अंत में आपके OS के अनुसार एक सेपरेटर (`/` या `\\`) हो।  
- **असमर्थित DWG संस्करण** – यह सुनिश्चित करें कि आप नवीनतम Aspose.CAD संस्करण उपयोग कर रहे हैं; पुराने DWG संस्करणों को कनवर्ट करना पड़ सकता है।  
- **मेमोरी उपयोग** – बड़े ड्रॉइंग्स काफी मेमोरी ले सकते हैं। समाप्त होने पर `CadImage` ऑब्जेक्ट को डिस्पोज़ करें: `cadImage.dispose();`।  
- **हेडलेस सर्वर पर चलाना** – कोई UI कॉम्पोनेन्ट आवश्यक नहीं है, इसलिए कोड Linux सर्वर पर बिना ग्राफ़िकल वातावरण के भी काम करता है।

## निष्कर्ष

बधाई हो! अब आप **DWG कैसे पढ़ें** और Aspose.CAD for Java का उपयोग करके उसके लेआउट्स की सूची बनाना जानते हैं। यह तकनीक अधिक उन्नत CAD ऑटोमेशन के लिए आधार बनती है, जैसे विशिष्ट लेआउट्स को इमेज, PDF में एक्सपोर्ट करना, या Linux पर DWG को PDF में बदलना। अधिक गहन जानकारी के लिए आधिकारिक [डॉक्यूमेंटेशन](https://reference.aspose.com/cad/java/) देखें।

**प्रश्न: क्या यह तरीका Linux पर DWG फ़ाइलें पढ़ने के लिए काम करता है?**  
उत्तर: बिल्कुल। चूँकि समाधान पूरी तरह से Java में है, यह किसी भी संगत JDK वाले OS पर चलता है।

**प्रश्न: क्या मैं पूरी ड्राइंग को मेमोरी में लोड किए बिना DWG फ़ाइल पढ़ सकता हूँ?**  
उत्तर: Aspose.CAD ड्राइंग को मेमोरी में लोड करता है; बहुत बड़ी फ़ाइलों के लिए आप उन्हें अलग‑थलग थ्रेड्स में प्रोसेस करने या भविष्य के रिलीज़ में उपलब्ध स्ट्रीमिंग API का उपयोग करने पर विचार कर सकते हैं।

**प्रश्न: क्या लेआउट्स को नाम के आधार पर फ़िल्टर किया जा सकता है?**  
उत्तर: हाँ – `CadLayoutDictionary` प्राप्त करने के बाद आप `layout.getLayoutName().equalsIgnoreCase("MyLayout")` की जाँच करके आवश्यक लेआउट को प्रोसेस कर सकते हैं।
