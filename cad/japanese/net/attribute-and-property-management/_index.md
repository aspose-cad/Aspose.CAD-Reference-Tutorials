## CADにおける「add custom properties」とは？

Adding custom properties means embedding user‑defined key/value pairs directly into a CAD file (DWG, DXF, etc.). These properties travel with the drawing, enabling downstream applications, BIM workflows, and collaboration tools to read and act on the metadata without altering the geometry.

## なぜ属性とプロパティを管理するのか？

- **Improved discoverability:** Search engines and file managers can index custom properties.  
- **Better collaboration:** Teams can see project‑level information (author, revision, client) at a glance.  
- **Automation‑ready:** Scripts and APIs can query properties to drive batch processing or reporting.  
- **Compliance:** Many industry standards require specific metadata fields; custom properties let you meet those requirements.

## よくある質問

**Q: Can I add both attributes and custom properties to the same drawing?**  
A: Yes. Attributes are attached to specific entities, while custom properties are stored at the file level. Using both gives you granular control and high‑level metadata.

**Q: Do custom properties affect file size?**  
A: The impact is negligible—typically a few kilobytes—since they are stored as simple key/value pairs.

**Q: Is there a limit to the number of properties I can add?**  
A: Practically no. Aspose.CAD does not impose a hard limit, but keep the list manageable for performance and maintainability.

**Q: Will these properties be visible in AutoCAD?**  
A: Yes. Custom properties appear in the “Properties” palette, and attributes can be displayed using the ATTEDIT command.

**Q: How do I read the properties back in my application?**  
A: Use the `CadDocument.CustomProperties` collection and the `Entity.Attributes` collection provided by Aspose.CAD.
