# CAD 导出选项

## 介绍

如果您是一名希望 **快速可靠地将 AutoCAD 导出为 PDF** 的 Java 开发者，那么您来对地方了。在本指南中，我们将逐步演示 Aspose.CAD for Java 最常见的导出场景——包括 AutoCAD 图像、CAD 布局、BMP、PDF、IFC 和 STL 文件。您将了解这些功能为何重要、它们在实际项目中的作用，以及可以直接复制到自己代码库中的一步步操作说明。

## 快速答案
- **导出 AutoCAD 为 PDF 的最简方法是什么？** 使用 Aspose.CAD 的 `Image.save("output.pdf", new PdfOptions())`。  
- **我可以将 IFC 文件转换为什么格式？** 完全支持 PNG；只需加载 IFC 文件并保存为 PNG。  
- **我能直接将 STL 文件导出为 PNG 吗？** 可以——加载 STL 模型后调用 `save("image.png")`。  
- **生产环境是否需要许可证？** 商业许可证是非试用部署的必需。  
- **需要哪个 Java 版本？** 推荐使用 Java 8 或更高版本。

## 什么是 **export autocad to pdf**？

将 AutoCAD 图纸导出为 PDF 意味着将基于矢量的 DWG/DXF 文件转换为一种被广泛接受的、面向页面的格式。PDF 能保留图层、线宽和颜色，非常适合与客户、审阅者或不支持原生 CAD 文件的下游系统共享设计。

## 为什么使用 Aspose.CAD for Java？

- **零安装，纯 Java** – 无需本地依赖或 COM 互操作。  
- **高保真** – 几何体、文本和光栅数据能够精确再现。  
- **批量处理** – 在单个循环中导出数十个文件，内存占用极低。  
- **广泛的格式支持** – 从经典的 DWG/DXF 到现代的 IFC 与 STL，全部使用统一的 API。

### 小技巧
- **专业提示：** 设置 `PdfOptions.setVectorRasterizationOptions()` 以控制光栅化导出的 DPI。  
- **避免内存激增：** 在处理大量文件时，每次保存后调用 `Image.dispose()`。  
- **故障排除：** 若图层消失，确保源 DWG 包含可见图层，并启用 `PdfOptions.setCompress(true)`。

## 常见问题

**问：如何使用 Aspose.CAD 将 IFC 转换为 PNG？**  
答：使用 `Image.load("model.ifc")` 加载 IFC 文件，然后调用 `save("model.png", new PngOptions())`。

**问：我能直接将 CAD 布局导出为 PDF，而无需先转换为图像吗？**  
答：可以——Aspose.CAD 在内部将布局视为图像，因此 `save("layout.pdf", new PdfOptions())` 会自动处理。

**问：导出 AutoCAD 为 PDF 与导出 CAD 布局为 PDF 有何区别？**  
答：导出 AutoCAD 图纸会转换整个图纸，而导出布局则针对特定的纸空间视图，保留其页面设置。

**问：导出多页 DWG 为 PDF 时，页面数量有限制吗？**  
答：没有固有限制；每个布局都会在生成的 PDF 中成为单独的一页。

**问：每种导出格式都需要单独的许可证吗？**  
答：单一的 Aspose.CAD 许可证覆盖所有受支持的格式（PDF、PNG、BMP 等）。
