# Μετρημένη Άδεια στο Aspose.CAD

## Εισαγωγή

Αποκτήστε το πλήρες δυναμικό του Aspose.CAD για Java με **how to set metered** άδεια! Αυτός ο οδηγός βήμα προς βήμα σας καθοδηγεί σε κάθε στάδιο — από την εγκατάσταση των προαπαιτούμενων, την εισαγωγή των σωστών πακέτων, μέχρι τη μέτρηση της κατανάλωσης πριν και μετά την επεξεργασία. Στο τέλος θα γνωρίζετε ακριβώς **how to set metered** κλειδιά, να διαβάζετε μετρικές χρήσης και να διατηρείτε τη ροή εργασίας CAD οικονομικά αποδοτική.

## Γρήγορες Απαντήσεις
- **Τι είναι η μετρημένη άδεια;** Ένα μοντέλο βάσει χρήσης που παρακολουθεί τις κλήσεις API και χρεώνει ανά μονάδα κατανάλωσης.  
- **Πόσα κλειδιά απαιτούνται;** Δύο κλειδιά – ένα δημόσιο και ένα ιδιωτικό – που δημιουργούνται από τον λογαριασμό σας στο Aspose.  
- **Μπορώ να ελέγξω τη χρήση προγραμματιστικά;** Ναι, καλέστε `License.getConsumptionQuantity()` πριν και μετά την επεξεργασία.  
- **Υπάρχει δοκιμαστική έκδοση;** Μπορείτε να αποκτήσετε δωρεάν δοκιμαστική άδεια από την ιστοσελίδα του Aspose.  
- **Ποια έκδοση της Java υποστηρίζεται;** Οι εκδόσεις Java 8 μέχρι 17 υποστηρίζονται πλήρως.

## Τι είναι η μετρημένη άδεια στο Aspose.CAD;
Η μετρημένη άδεια είναι το μοντέλο pay‑as‑you‑go του Aspose.CAD που καταγράφει κάθε κλήση API ως μονάδα κατανάλωσης. Σας επιτρέπει να παρακολουθείτε ακριβώς τη χρήση, να αποφεύγετε την υπερπρομήθεια και να πληρώνετε μόνο για ό,τι καταναλώνετε πραγματικά.

## Γιατί να χρησιμοποιήσετε μετρημένη άδεια για επεξεργασία CAD;
Το Aspose.CAD υποστηρίζει **50+** μορφές εισόδου και εξόδου — συμπεριλαμβανομένων των DWG, DXF, DGN και SVG — και μπορεί να επεξεργαστεί αρχεία έως **2 GB** χωρίς να φορτώνει ολόκληρο το έγγραφο στη μνήμη. Αυτή η αποδοτικότητα μεταφράζεται σε χαμηλότερο κόστος διακομιστή, ειδικά όταν διαχειρίζεστε μεγάλες παρτίδες σχεδίων.

## Πώς να ορίσετε μετρημένη άδεια στο Aspose.CAD για Java;

Φορτώστε τα δημόσια και ιδιωτικά σας κλειδιά, καλέστε `License.setMeteredKey(publicKey, privateKey)`, και η βιβλιοθήκη μεταβαίνει αμέσως σε λειτουργία μετρημένης άδειας. Αυτή η ενιαία κλήση ενεργοποιεί την παρακολούθηση χρήσης για κάθε επόμενη λειτουργία API, επιτρέποντάς σας να ερωτήσετε την κατανάλωση πριν και μετά από οποιοδήποτε βήμα επεξεργασίας.

## Εισαγωγή Πακέτων

Για να χρησιμοποιήσετε τη βιβλιοθήκη, εισάγετε το κύριο πακέτο της:

`import com.aspose.cad.*;` φέρνει τις κλάσεις Aspose.CAD στο έργο σας.

```java
import com.aspose.cad;
```

## Βήμα 1: Ορισμός Μετρημένου Κλειδιού

`setMeteredKey` καταχωρεί τα δημόσια και ιδιωτικά σας κλειδιά στη βιβλιοθήκη Aspose.CAD.

Αρχικά, ορίστε το μετρημένο κλειδί χρησιμοποιώντας τη μέθοδο `setMeteredKey`. Αντικαταστήστε `<valid public key>` και `<valid private key>` με τα πραγματικά σας δημόσια και ιδιωτικά κλειδιά.

```java
Metered.setMeteredKey("<valid public key>", "<valid private key>");
```

## Βήμα 2: Λήψη Ποσότητας Κατανάλωσης Πριν την Επεξεργασία

`getConsumptionQuantity` επιστρέφει το συνολικό αριθμό μονάδων κατανάλωσης που έχουν καταγραφεί μέχρι τώρα.

Ανακτήστε την τιμή της καταναλωμένης ποσότητας πριν την πρόσβαση στο API του Aspose.CAD για να αποκτήσετε μια αρχική κατανόηση.

```java
BigDecimal quantityOld = Metered.getConsumptionQuantity();
System.out.println("Consumption quantity before processing: " + quantityOld);
```

## Βήμα 3: Επεξεργασία

Εκτελέστε την επιθυμητή επεξεργασία CAD χρησιμοποιώντας τις λειτουργίες του Aspose.CAD. Αποσχολιάστε το απόσπασμα κώδικα που σχετίζεται με τη συγκεκριμένη εργασία σας, όπως η φόρτωση μιας εικόνας CAD.

```java
// Example:
// com.aspose.cad.fileformats.cad.CadImage image =
// (com.aspose.cad.fileformats.cad.CadImage)com.aspose.cad.Image.load("BlockRefDgn.dwg");
```

## Βήμα 4: Λήψη Ποσότητας Κατανάλωσης Μετά την Επεξεργασία

Μετά την επεξεργασία, λάβετε ξανά την τιμή της καταναλωμένης ποσότητας για να αξιολογήσετε την επίδραση.

```java
BigDecimal quantity = Metered.getConsumptionQuantity();
System.out.println("Consumption quantity after processing: " + quantity);
```

Επαναλάβετε αυτά τα βήματα για οποιαδήποτε επιπλέον επεξεργασία ή εργασίες, όπως απαιτείται.

## Κοινά Προβλήματα και Λύσεις

- **Σφάλμα μη έγκυρου κλειδιού:** Ελέγξτε ξανά ότι και τα δύο κλειδιά έχουν αντιγραφεί ακριβώς όπως εμφανίζονται στον λογαριασμό σας στο Aspose· τα επιπλέον κενά προκαλούν αποτυχία πιστοποίησης.  
- **Αναφέρεται μηδενική κατανάλωση:** Βεβαιωθείτε ότι καλείτε `License.getConsumptionQuantity()` *μετά* την πρώτη χρήση του API· η πρώτη κλήση αρχικοποιεί το μετρητή.
