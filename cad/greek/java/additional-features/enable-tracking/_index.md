# Εξαγωγή DWG σε PDF και Ενεργοποίηση Παρακολούθησης με Aspose.CAD Java

## Εισαγωγή

Η εξαγωγή DWG σε PDF διατηρώντας πλήρη ορατότητα στη διαδικασία μετατροπής είναι απαραίτητη για τις σύγχρονες ομάδες που εστιάζουν στο CAD. Σε αυτόν τον οδηγό θα ανακαλύψετε **πώς να ενεργοποιήσετε την παρακολούθηση** στις ροές εργασίας DWG, να μετατρέψετε DXF σε PDF στην ίδια αλυσίδα, και να καταγράψετε κάθε προειδοποίηση απόδοσης με μια **προσαρμοσμένη κλάση διαχείρισης σφαλμάτων Java**. Στο τέλος θα έχετε ένα έτοιμο για παραγωγή απόσπασμα κώδικα που όχι μόνο παράγει PDF υψηλής πιστότητας αλλά επίσης καταγράφει διαγνωστικές πληροφορίες για έλεγχο και αντιμετώπιση προβλημάτων.

## Γρήγορες Απαντήσεις
- **Τι κάνει το “enable dwg tracking java”;** Ενεργοποιεί τις κλήσεις επιστροφής απόδοσης του Aspose.CAD ώστε να μπορείτε να δείτε προειδοποιήσεις και σφάλματα κατά την εξαγωγή.  
- **Χρειάζομαι άδεια;** Μια δοκιμαστική έκδοση λειτουργεί για δοκιμές· απαιτείται εμπορική άδεια για χρήση σε παραγωγή.  
- **Μπορώ επίσης να μετατρέψω DXF σε PDF;** Ναι – το ίδιο αντικείμενο `PdfOptions` που χρησιμοποιείται για DWG λειτουργεί και για DXF, καλύπτοντας το σενάριο *java convert dxf pdf*.  
- **Ποια έκδοση JDK απαιτείται;** Java 8 ή νεότερη.  
- **Πού μπορώ να βρω περισσότερα παραδείγματα;** Δείτε την [Τεκμηρίωση Aspose.CAD Java](https://reference.aspose.com/cad/java/) παρακάτω.

## Τι είναι η εξαγωγή DWG σε PDF;
Η εξαγωγή DWG σε PDF είναι η διαδικασία μετατροπής ενός εγγενούς σχεδίου AutoCAD (DWG) σε ένα φορητό έγγραφο PDF, διατηρώντας την ακρίβεια των διανυσματικών στοιχείων, τα επίπεδα και τις σημειώσεις. Το Aspose.CAD εκτελεί αυτή τη μετατροπή εξ ολοκλήρου σε Java, εξαλείφοντας την ανάγκη για εγκαταστάσεις του AutoCAD.

## Γιατί να χρησιμοποιήσετε Aspose.CAD για Java;
Το Aspose.CAD για Java παρέχει μια ολοκληρωμένη, καθαρά‑Java λύση για μετατροπή CAD, υποστηρίζοντας πάνω από 50 μορφές, προσφέροντας υψηλή απόδοση και παρέχοντας ενσωματωμένη παρακολούθηση μέσω κλήσεων επιστροφής απόδοσης. Δεν απαιτεί εξωτερικές εξαρτήσεις και μπορεί να ενσωματωθεί σε αλυσίδες διακομιστή.

- **Πλήρης υποστήριξη μορφών** – χειρίζεται DWG, DXF, DGN και 50+ άλλες μορφές CAD.  
- **Καμία εξωτερική εξάρτηση** – βιβλιοθήκη καθαρά‑Java ιδανική για αυτοματοποίηση στο διακομιστή.  
- **Ενσωματωμένη παρακολούθηση** – `CadRenderHandler` παρέχει λεπτομερή αποτελέσματα απόδοσης.  
- **Εξαγωγή PDF με μία γραμμή** – `PdfOptions` μετατρέπει σε PDF διατηρώντας τα διανυσματικά δεδομένα αμετάβλητα.  

Αυτές οι ποσοτικοποιημένες δηλώσεις υποστηρίζονται από την ικανότητα του Aspose.CAD να επεξεργάζεται αρχεία έως 500 σελίδες χωρίς να φορτώνει ολόκληρο το έγγραφο στη μνήμη, επιτυγχάνοντας χρόνους μετατροπής κάτω από 2 δευτερόλεπτα σε έναν τυπικό διακομιστή 4‑πυρήνων.

## Πώς να εξάγετε DWG σε PDF;  

Φορτώστε το αρχείο προέλευσης, διαμορφώστε το `PdfOptions`, συνδέστε έναν προσαρμοσμένο `CadRenderHandler` και καλέστε το `save`. Αυτή η ροή από άκρο σε άκρο μετατρέπει DWG (ή DXF) σε PDF ενώ εκδίδει πληροφορίες παρακολούθησης για κάθε βήμα απόδοσης, διασφαλίζοντας ότι τυχόν προειδοποιήσεις ή σφάλματα καταγράφονται και μπορούν να ληφθούν μέτρα από προγραμματιστές ή αυτοματοποιημένες διαδικασίες.

## Εισαγωγή Namespaces

Η κλάση `CadRenderHandler` είναι η διεπαφή κλήσης επιστροφής του Aspose.CAD που λαμβάνει διαγνωστικά απόδοσης. Εισάγετε τα απαιτούμενα πακέτα πριν ξεκινήσετε τον κώδικα:

```java
import com.aspose.cad.Image;

import com.aspose.cad.imageoptions.CadRasterizationOptions;
import com.aspose.cad.imageoptions.CadRenderResult;
import com.aspose.cad.imageoptions.PdfOptions;
import com.aspose.cad.imageoptions.RenderResult;
import java.io.FileNotFoundException;
import java.io.FileOutputStream;
import java.io.OutputStream;
```

## Βήμα 1: Φόρτωση του αρχείου DWG/DXF  

Η κλάση `CadImage` αντιπροσωπεύει ένα έγγραφο CAD που φορτώνεται στη μνήμη. Η δημιουργία της με διαδρομή αρχείου φορτώνει το σχέδιο χωρίς να ανοίγει μια εγγενή εφαρμογή CAD.

```java
String dataDir = "Your Document Directory" + "DXFDrawings/";
Image image = Image.load(dataDir + "conic_pyramid.dxf");
```

## Βήμα 2: Διαμόρφωση Επιλογών Εξαγωγής PDF (Πώς να Μετατρέψετε DXF σε PDF)

`PdfOptions` ορίζει πώς ο rasterizer CAD πρέπει να παράγει την έξοδο PDF, συμπεριλαμβανομένων των ρυθμίσεων διανυσματικής απόδοσης και του μεγέθους σελίδας. Η ρύθμιση του `VectorRasterizationOptions` εξασφαλίζει ότι οι γραμμές και οι καμπύλες παραμένουν καθαρές. Το αντικείμενο `VectorRasterizationOptions` καθορίζει παραμέτρους rasterization όπως DPI και χρώμα φόντου.

```java
OutputStream stream = new FileOutputStream(dataDir + "output_conic_pyramid.pdf");
PdfOptions pdfOptions = new PdfOptions();
CadRasterizationOptions cadRasterizationOptions = new CadRasterizationOptions();
pdfOptions.setVectorRasterizationOptions(cadRasterizationOptions);
cadRasterizationOptions.setPageWidth(800);
cadRasterizationOptions.setPageHeight(600);
```

## Βήμα 3: Υλοποίηση Παρακολούθησης (Προσαρμοσμένη Διαχείριση Σφαλμάτων Java)

`ErrorHandler` επεκτείνει το `CadRenderHandler` για να καταγράψει προειδοποιήσεις, σφάλματα και πληροφοριακά μηνύματα που εκδίδονται κατά τη rasterization. Αυτή η κλάση γράφει κάθε μήνυμα στην κονσόλα, αλλά μπορείτε εύκολα να τα κατευθύνετε σε αρχείο καταγραφής ή σύστημα παρακολούθησης. Το `CadRenderHandler` είναι μια διεπαφή που λαμβάνει διαγνωστικά απόδοσης από τον rasterizer.

```java
cadRasterizationOptions.RenderResult = new ErrorHandler();
```

## Βήμα 4: Εξαγωγή σε PDF  

Καλώντας το `save` στο αντικείμενο `CadImage` με τις διαμορφωμένες `PdfOptions` εκτελεί τη μετατροπή. Επειδή ο προσαρμοσμένος χειριστής είναι ήδη συνδεδεμένος, τυχόν προβλήματα απόδοσης εμφανίζονται στην κονσόλα καθώς εκτελείται η εξαγωγή.

```java
System.out.println("Exporting to pdf format");
image.save(stream, pdfOptions);
```

## Βήμα 5: Κλάση CadRenderHandler  

`CadRenderHandler` είναι η βασική κλάση του Aspose.CAD για λήψη αποτελεσμάτων απόδοσης. Η υπερφόρτωση της μεθόδου `onRenderCompleted` σας δίνει πρόσβαση στο αντικείμενο `RenderResult`, το οποίο περιέχει μια συλλογή από καταχωρήσεις `RenderMessage` που περιγράφουν κάθε βήμα της διαδικασίας.

```java
public static class ErrorHandler extends CadRasterizationOptions.CadRenderHandler {
    @Override
    public void invoke(CadRenderResult result) {
        System.out.println("Tracking results of exporting");

        if (result.isRenderComplete())
            return;

        System.out.println("Have some problems:");

        int idxError = 1;
        for (RenderResult rr : result.getFailures()) {
            System.out.printf("{0}. {1}, {2}", idxError, rr.getRenderCode(), rr.getMessage());
            idxError++;
        }
    }
}
```

Η ενεργοποίηση της παρακολούθησης δημιουργεί ένα ίχνος ελέγχου που ικανοποιεί τις απαιτήσεις συμμόρφωσης σε κανονιστικούς τομείς όπως η αρχιτεκτονική, η μηχανική και η κατασκευή. Η προσαρμοσμένη διαχείριση σφαλμάτων σας επιτρέπει επίσης να ενσωματώσετε ελέγχους υγείας μετατροπής CAD σε αλυσίδες CI/CD, σημαδεύοντας αυτόματα αρχεία που χρειάζονται χειροκίνητη αξιολόγηση.

## Συχνά Προβλήματα και Λύσεις

- **`NullPointerException` στο `RenderResult`** – Βεβαιωθείτε ότι χρησιμοποιείτε Aspose.CAD 23.10 ή νεότερο· οι παλαιότερες εκδόσεις δεν διαθέτουν το API `RenderResult`.  
- **Αρχείο δεν βρέθηκε** – Ελέγξτε ξανά ότι το `dataDir` δείχνει στον σωστό φάκελο και ότι το όνομα αρχείου ταιριάζει ακριβώς με τη διάκριση πεζών‑κεφαλαίων.  
- **Απουσία εξόδου παρακολούθησης** – Επιβεβαιώστε ότι η παρουσία `ErrorHandler` έχει ανατεθεί σωστά στο `cadRasterizationOptions.setRenderHandler(new ErrorHandler())`.
