# แปลง DWG เป็น PDF (PDF/A1a & A1b) ด้วย Aspose.CAD สำหรับ Java

## บทนำ

ในกระบวนการทำงานด้านวิศวกรรมและการออกแบบสมัยใหม่ **การแปลง DWG เป็น PDF** เป็นความต้องการประจำวันสำหรับการแชร์, การเก็บรักษา, และการปฏิบัติตามรูปแบบเอกสารมาตรฐานอุตสาหกรรม. PDF/A‑1a และ PDF/A‑1b เป็นมาตรฐานการเก็บถาวรที่ได้รับการยอมรับอย่างกว้างขวางที่สุด, รับประกันว่าการวาดของคุณจะเรนเดอร์แบบเดียวกันบนทุกระบบ. Aspose.CAD สำหรับ Java ทำให้การแปลงนี้เป็นเรื่องง่าย, เชื่อถือได้, และสามารถโปรแกรมได้เต็มที่. ในบทเรียนนี้คุณจะได้เรียนรู้วิธีแปลงไฟล์ DWG เป็น PDF ที่สอดคล้องกับ PDF/A ด้วยเพียงไม่กี่บรรทัดของโค้ด Java.

## คำตอบสั้น
- **ไลบรารีที่จัดการการแปลงคืออะไร?** Aspose.CAD for Java  
- **มาตรฐาน PDF/A ที่รองรับคืออะไร?** PDF/A‑1a และ PDF/A‑1b  
- **ฉันต้องใช้ไลเซนส์สำหรับการทดสอบหรือไม่?** ใช่ – มีไลเซนส์ชั่วคราวให้จาก Aspose.  
- **เวอร์ชัน Java ขั้นต่ำคืออะไร?** แนะนำให้ใช้ Java 8 หรือสูงกว่า.  
- **ฉันสามารถประมวลผลหลายไฟล์ DWG เป็นชุดได้หรือไม่?** แน่นอน – ทำซ้ำขั้นตอนสำหรับแต่ละไฟล์หรือวนลูปผ่านโฟลเดอร์.

## “การแปลง DWG เป็น PDF” คืออะไรและทำไมจึงสำคัญ?
การแปลงภาพวาด DWG เป็น PDF สร้างเอกสารที่สามารถดูได้ทั่วโลกในขณะที่ยังคงรักษาความแม่นยำของเวกเตอร์. เมื่อคุณเลือกความสอดคล้องกับ PDF/A‑1a หรือ PDF/A‑1b, ไฟล์ยังฝังโปรไฟล์สี, ฟอนต์, และเมตาดาต้าที่จำเป็นสำหรับการเก็บถาวรระยะยาว, ซึ่งเป็นสิ่งสำคัญสำหรับการส่งเอกสารตามกฎระเบียบ, เอกสารก่อสร้าง, และการส่งมอบให้ลูกค้า.

## ทำไมต้องใช้ Aspose.CAD สำหรับ Java?
- **รองรับเวอร์ชัน DWG ทั้งหมด** – ตั้งแต่ R12 รุ่นเก่าไปจนถึงรุ่นล่าสุด.  
- **ไม่มีการพึ่งพาภายนอก** – ไลบรารีทำงานได้ทันที, ไม่ต้องใช้ซอฟต์แวร์ CAD.  
- **การควบคุมละเอียด** – คุณสามารถตั้งค่าตัวเลือกการเรสเตอร์ไลซ์, DPI, ขนาดหน้า, และการปฏิบัติตาม PDF/A.  
- **ประสิทธิภาพสูง** – เหมาะสำหรับการประมวลผลเป็นชุดหลายพันภาพวาด.

## ข้อกำหนดเบื้องต้น

ก่อนเริ่ม, โปรดตรวจสอบว่าคุณมี:

- สภาพแวดล้อมการพัฒนา Java (JDK 8 หรือใหม่กว่า) พร้อม IDE ที่คุณชื่นชอบ.  
- ไลบรารี Aspose.CAD สำหรับ Java – ดาวน์โหลดจาก [ลิงก์ดาวน์โหลด](https://releases.aspose.com/cad/java/).  
- โฟลเดอร์ที่มีไฟล์ DWG ที่คุณต้องการแปลง.

## นำเข้า Namespaces

ก่อนอื่น, ให้นำเข้าคลาสที่คุณต้องการ. การวาง import ไว้ที่ส่วนบนของไฟล์ทำให้โค้ดอ่านง่ายและบำรุงรักษาได้ดี.

```java
import com.aspose.cad.Image;

import com.aspose.cad.imageoptions.CadRasterizationOptions;
import com.aspose.cad.imageoptions.PdfCompliance;
import com.aspose.cad.imageoptions.PdfDocumentOptions;
import com.aspose.cad.imageoptions.PdfOptions;
```

## ขั้นตอนที่ 1: ตั้งค่าไดเรกทอรีทรัพยากร

กำหนดเส้นทางที่ไฟล์ DWG ของคุณอยู่. ปรับสตริงให้ชี้ไปยังไดเรกทอรีจริงของคุณ.

```java
String dataDir = "Your Document Directory" + "DWGDrawings/";
```

## ขั้นตอนที่ 2: โหลดไฟล์ DWG

ใช้ `Image.load` เพื่ออ่านไฟล์ DWG เข้าหน่วยความจำ. วัตถุนี้จะถูกบันทึกเป็น PDF ต่อไป.

```java
String srcFile = dataDir + "Bottom_plate.dwg";
Image objImage = Image.load(srcFile);
```

## ขั้นตอนที่ 3: สร้าง PDF Options

สร้างอินสแตนซ์ของ `PdfOptions` และแนบอ็อบเจกต์ `CadRasterizationOptions`. ตัวเลือกการเรสเตอร์ไลซ์ช่วยให้คุณควบคุมวิธีการเรนเดอร์ข้อมูลเวกเตอร์ภายใน PDF.

```java
PdfOptions pdfOptions = new PdfOptions();
pdfOptions.setVectorRasterizationOptions(new CadRasterizationOptions());
```

## ขั้นตอนที่ 4: ตั้งค่า Core PDF Options (Compliance)

ที่นี่คุณบอก Aspose.CAD ว่าต้องการระดับการปฏิบัติตาม PDF/A ใด. ตัวอย่างเริ่มต้นด้วย PDF/A‑1a.

```java
pdfOptions.setCorePdfOptions(new PdfDocumentOptions());
pdfOptions.getCorePdfOptions().setCompliance(PdfCompliance.PdfA1a);
```

## ขั้นตอนที่ 5: บันทึก PDF ด้วยการปฏิบัติตาม PDF/A‑1a

ตอนนี้ให้เขียนไฟล์ PDF ลงดิสก์. ไฟล์ผลลัพธ์จะสอดคล้องกับ PDF/A‑1a, พร้อมสำหรับการเก็บถาวร.

```java
objImage.save(dataDir + "Saved1.pdf", pdfOptions);
```

## ขั้นตอนที่ 6: เปลี่ยนการปฏิบัติตามเป็น PDF/A‑1b และบันทึกอีกครั้ง

หากต้องการ PDF/A‑1b เพียงสลับแฟล็กการปฏิบัติตามและบันทึกไฟล์ที่สอง.

```java
pdfOptions.getCorePdfOptions().setCompliance(PdfCompliance.PdfA1b);
objImage.save(dataDir + "Saved.pdf", pdfOptions);
```

> **เคล็ดลับ:** ห่อหุ้มตรรกะการแปลงในเมธอดที่ใช้ซ้ำได้เพื่อให้คุณเรียกใช้สำหรับทุก DWG ในโฟลเดอร์, ลดโค้ดซ้ำและเพิ่มการบำรุงรักษา.

## กรณีการใช้งานทั่วไป

| สถานการณ์ | ทำไมต้อง PDF/A? |
|----------|-----------------|
| **การส่งเอกสารตามกฎระเบียบ** | รับประกันการอ่านได้ในระยะยาวและความถูกต้องตามกฎหมาย. |
| **ผลผลิตสำหรับลูกค้า** | ลูกค้าสามารถเปิดไฟล์ได้โดยไม่ต้องใช้ซอฟต์แวร์ CAD. |
| **ระบบจัดการเอกสาร** | ไฟล์ PDF/A สามารถทำดัชนีและค้นหาได้ในแพลตฟอร์ม DMS ส่วนใหญ่. |

## ปัญหาทั่วไปและวิธีแก้

- **ฟอนต์หรือสัญลักษณ์หาย** – ตรวจสอบให้แน่ใจว่าไฟล์ DWG ฝังฟอนต์ที่ต้องการทั้งหมด, หรือตั้งค่า `CadRasterizationOptions.setEmbedFonts(true)`.  
- **ขนาดไฟล์ใหญ่** – ลด DPI ในตัวเลือกการเรสเตอร์ไลซ์หรือเปิดการบีบอัดด้วย `PdfDocumentOptions.setCompress(true)`.  
- **ไม่พบไลเซนส์** – ใช้ไลเซนส์ชั่วคราวหรือถาวรก่อนการแปลง; หากไม่ทำจะได้รับลายน้ำ.
