# แยกแยะรูปแบบ DWT DWG – คู่มือ Aspose.CAD

## บทนำ

เมื่อคุณทำงานกับโครงการที่ใช้ AutoCAD ความสามารถในการ **แยกแยะรูปแบบ DWT และ DWG** จะช่วยประหยัดเวลาและป้องกันข้อผิดพลาดในการแปลงที่มีค่าใช้จ่ายสูง ไม่ว่าคุณจะสร้างเครื่องมือประมวลผลแบบชุดหรือเพียงแค่ต้องการตรวจสอบไฟล์ที่เข้ามา การรู้ประเภทไฟล์ที่แน่นอนจะทำให้คุณสามารถกำหนดเส้นทางไฟล์แต่ละไฟล์ไปยังเวิร์กโฟลว์ที่เหมาะสมได้ ในบทแนะนำนี้เราจะแสดงให้คุณเห็นขั้นตอน‑ต่อ‑ขั้นตอนว่าใช้ **Aspose.CAD for .NET** เพื่อระบุไฟล์ DWT และ DWG อย่างโปรแกรมเมติก

## คำตอบสั้น
- **ไลบรารีใดที่ระบุรูปแบบ CAD?** Aspose.CAD for .NET  
- **เมธอดใดที่คืนค่ารูปแบบไฟล์?** `Image.GetFileFormat`  
- **รูปแบบที่รองรับสำหรับการตรวจจับ?** DWT, DWG, DXF, และอื่น ๆ อีกมาก  
- **ฉันต้องการไลเซนส์สำหรับการทดสอบหรือไม่?** ทดลองใช้งานฟรีได้; ต้องมีไลเซนส์สำหรับการใช้งานจริง

## อะไรคือ “distinguish dwt dwg”?

“Distinguish dwt dwg” หมายถึงการตรวจจับว่าไฟล์ CAD ที่กำหนดเป็น **Drawing Template (DWT)** หรือ **Drawing (DWG)** DWT ทำหน้าที่เป็นแม่แบบที่มีเลเยอร์, สไตล์และการตั้งค่าที่กำหนดไว้ล่วงหน้า ส่วน DWG จะเก็บข้อมูลการวาดจริง การแยกแยะเหล่านี้ตั้งแต่ต้นในไพป์ไลน์ของคุณช่วยให้คุณสามารถใช้กฎการประมวลผลที่เหมาะสมได้

## ทำไมต้องแยกแยะรูปแบบ DWT และ DWG?

- **Automation:** กำหนดเส้นทางแม่แบบไปยังระบบจัดการแม่แบบโดยอัตโนมัติและกำหนดเส้นทางการวาดไปยังเอนจินการเรนเดอร์  
- **Compliance:** บังคับใช้มาตรฐานของบริษัทโดยการปฏิเสธรูปแบบที่ไม่รองรับก่อนที่ไฟล์จะเข้าสู่คลังข้อมูลของคุณ  
- **Performance:** ข้ามขั้นตอนการแปลงที่ไม่จำเป็นสำหรับไฟล์ที่อยู่ในรูปแบบที่ต้องการแล้ว

## ขั้นตอน 1: ตรวจสอบรูปแบบ DWT

### 1.1 โหลดไฟล์ DWT

```csharp
// Load the DWT file
var formatTypeDwt = Image.GetFileFormat(GetFileFromDesktop("sample.dwt"));
```

### 1.2 ตรวจสอบรูปแบบ

```csharp
// Verify if the format is DWT
Assert.IsTrue(formatTypeDwt.ToString().ToLower().Contains("dwt"));
```

> **เคล็ดลับมืออาชีพ:** `GetFileFormat` ทำงานได้กับเส้นทางไฟล์หรือสตรีม ดังนั้นคุณสามารถรวมเข้ากับเว็บเซอร์วิสที่รับการอัปโหลดได้

## ขั้นตอน 2: ตรวจสอบรูปแบบ DWG

### 2.1 โหลดไฟล์ DWG

```csharp
// Load the DWG file
var formatTypeDwg = Image.GetFileFormat(GetFileFromDesktop("sample.dwg"));
```

### 2.2 ตรวจสอบรูปแบบ

```csharp
// Verify if the format is DWG
Assert.IsTrue(formatTypeDwg.ToString().ToLower().Contains("dwg"));
```

> **ข้อผิดพลาดทั่วไป:** การลืมเรียก `.ToLower()` อาจทำให้เกิดปัญหาเรื่องความแตกต่างของตัวอักษรบนระบบปฏิบัติการบางประเภท

ทำซ้ำสองขั้นตอนนี้สำหรับไฟล์เพิ่มเติมที่คุณต้องการวิเคราะห์ หลังจากตรวจสอบแล้วคุณจะสามารถ **แยกแยะรูปแบบ DWT และ DWG** ในแอปพลิเคชันของคุณได้อย่างมั่นใจ

## คำถามที่พบบ่อยเพิ่มเติม

**Q: การตรวจจับรูปแบบทำงานกับสตรีมแทนเส้นทางไฟล์หรือไม่?**  
A: แน่นอน `Image.GetFileFormat` รองรับ `Stream` ทำให้คุณตรวจจับรูปแบบโดยตรงจากหน่วยความจำหรือแหล่งเครือข่าย

**Q: สามารถตรวจจับรูปแบบ CAD อื่น ๆ ด้วยเมธอดเดียวกันได้หรือไม่?**  
A: ได้ เมธอดเดียวกันสามารถระบุ DXF, DGN, STL และรูปแบบอื่น ๆ ที่รองรับ – เพียงตรวจสอบสตริงที่คืนค่าว่ามีคีย์เวิร์ดที่ต้องการหรือไม่

**Q: มีผลต่อประสิทธิภาพเมื่อเช็คไฟล์ขนาดใหญ่หรือไม่?**  
A: การตรวจจับอ่านเฉพาะส่วนหัวของไฟล์ ดังนั้นแม้ไฟล์หลายเมกะไบต์ก็จะประมวลผลในระดับมิลลิวินาที

**Q: จำเป็นต้องทำการ Dispose วัตถุหลังจากเรียก `GetFileFormat` หรือไม่?**  
A: เมธอดนี้ไม่ถือทรัพยากรที่ไม่ได้จัดการ แต่หากคุณเปิด `FileStream` ด้วยตนเอง อย่าลืมปิดหรือ Dispose มัน

**Q: จะจัดการไฟล์ที่มีนามสกุลไม่ทราบได้อย่างไร?**  
A: เรียก `GetFileFormat` ไม่ว่ามีนามสกุลอะไร ไลบรารีจะกำหนดประเภทโดยอิงจากเนื้อหา ไม่ใช่ชื่อไฟล์
